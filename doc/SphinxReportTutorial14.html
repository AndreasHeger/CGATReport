Plotting from dataframe is very conveniently done with the ggplot package. First, we neet to import ggplot.









In [9]:





from ggplot import *













We can now try a simple graph.









In [12]:





ggplot( aes( x='bin', y='none'), data=result ) + geom_point()





























    Out[12]:






<ggplot: (10649645)>


















Our first plot is not very informative - we can do better. Let us colour the data points by experiremnt or 'track'. Currently, this information is part of the hierarchical index of the dataframe. For ggplot to be able to make use of that information we need to add this information into an additional column. The pandas.Dataframe.reset_index method does exactly that:









In [17]:





result.reset_index( 0, inplace = True )
result.columns
















    Out[17]:






Index([u'level_0', u'index', u'slice', u'track', u'bin', u'region', u'region_bin', u'none', u'area', u'counts', u'background'], dtype='object')


















Note how there is now a new column called 'track'. We can use the track information to annotate the plot.









In [20]:





ggplot( aes( x='bin', y='none', color='track'), data=result ) + geom_point()





























    Out[20]:






<ggplot: (9226849)>


















Much better. We notice that the profiles are not normalized for read-depth and are thus not directly comparable. So let us try plotting the normalized profiles. First, we plot profiles normalized by area.









In [23]:





ggplot( aes( x='bin', y='area', color='track'), data=result ) + geom_point()





























    Out[23]:






<ggplot: (9319297)>


















We can see that the background levels still differ between profiles. What has happenden is that even profiles will be close to an average, while profiles with peaks will depress the background. Let us try background normalization instead, which uses the profile densities at the extreme ends for normalization.









In [26]:





ggplot( aes( x='bin', y='background', color='track'), data=result ) + geom_point()





























    Out[26]:






<ggplot: (8323817)>


















This normalization allows us to compare profiles between experiments with different read dephts. We now want to put the plot above into our sphinx-report report. We can use the ggplot renderer. Note that this renderer automatically adds the track information as a column. Thus, we only need to take note of the parameters for the plot, the aesthatics and the geometry


    

  • aes: x='bin', y='background', color='track'
    • 

  • geom: geom_point()
    • 



We can now go back to the command line to run sphinxreport-test:


sphinxreport-test -r ggplot -t TrackerDataframes -o glob="data/*.tsv.gz" -o aes='x="bin", y="background", color="track"'
                  -o geom="geom_point()" -o regex="data/(.*).transcript*"
